git 与  github 的区别

git 和 github是2个不同的东西。
git 说白了就是版本控制器工具  主要用作项目代码更新、删改、还原（回滚）等。
github 是一个网站  算是一种工程师用的社交平台  里边有别人共享的项目  写的一些心得  或项目用作远程储存的仓库（服务器等可理解为网上不要钱的U盘）。




svn 与  git 的区别

svn  为集成式  （要进行版本控制一定要联网【要经过中央处理器才能进行版本控制】），比较依赖中央服务器
git  分布式  （ 不用网络就能进行版本控制，在本机进行版本控制即可 ）




git分为3个区域
分别为工作区域、暂存区域、版本区域
工作区域 :即项目在本地的那个文件夹的范围成为工作区域
暂存区域 :用作工作和版本区域的过渡  避免操作失误  保护两个区域
版本区域 :即做好的项目或者代码  可用于上传到github上便于别人down下来学习参考或添加


注意   在工作区域    为红色        在暂存区域     为绿色


简单的操作密令：

cd  
为查询、进入   比如进入d盘  则   cd d:

ls or ll
为展示、显示   显示某个文件夹下边的所有内容

clear
清楚屏幕内容

git status
查看状态    主要查看3个区域的内容

git log
查看版本号  版本号可以用作还原等

git add 文件名
把工作区的内容提交给暂存区  （文件名可以是文件夹  或者 txt、mp3等）

git add .
把工作区所有的内容提交给暂存区域   

git commit -m “注释”
把暂存区域的文件加上相对应的注释   提交给版本区域

git commit -a -m “注释”
把工作区域内部所有的东西提交给版本区域内部
注意 新建的文件不要要add后才可以   此为特例!

git push origin master
把版本区域内部的东西提交给github

假如上传几个相关联的文件  由于疏忽忘记push1个到多个 又想用同一个版本号那么可以
git commit -m “注释” --amend
